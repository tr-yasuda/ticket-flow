# Routing Foundation Design

Date: 2026-06-17
Issue: #123
Scope: `apps/web`

## Goal

Introduce a typed routing foundation for the web app using TanStack Router. Separate public routes (`/login`, `/signup`) from protected routes (`/app`, `/app/:organizationId/tickets`). Place the existing `AppShell` component under the protected route layout. Provide a `NotFound` page for undefined paths and ensure `organizationId` URL param is type-safe.

## Decisions

- **Router style**: File-based routing via `@tanstack/router-plugin/vite`.
  - Rationale: issue explicitly values typed URL params; file-based routing gives automatic param typing with minimal boilerplate.
- **Auth guard**: out of scope. Protected routes are only grouped under a shared layout where a guard can be added later.
- **Root path `/`**: redirect to `/login` because the acceptance criteria does not define a root page.
- **Placeholder pages**: minimal page components will be created under `src/pages` because the actual Login / Signup / Ticket screens are handled in separate issues.

## File Structure

```text
apps/web/src/
  main.tsx                          # mount RouterProvider
  routeTree.gen.ts                  # generated by TanStack Router plugin; committed so typecheck passes without a build step
  routes/
    __root.tsx                      # root layout (ThemeProvider, Toaster)
    index.tsx                       # / -> /login redirect
    login.tsx                       # public /login
    signup.tsx                      # public /signup
    app/
      route.tsx                     # protected layout: AppShell + Outlet
      index.tsx                     # /app
      $organizationId/
        tickets.tsx                 # /app/:organizationId/tickets
  pages/
    NotFoundPage.tsx
    LoginPage.tsx
    SignupPage.tsx
    AppTopPage.tsx
    OrganizationTicketsPage.tsx
  components/layout/AppShell.tsx    # existing, unchanged
```

## Route Tree

| Path | Layout / Component | Notes |
|------|--------------------|-------|
| `/` | redirect to `/login` | no dedicated root page |
| `/login` | `LoginPage` | public, no AppShell |
| `/signup` | `SignupPage` | public, no AppShell |
| `/app` | `AppShell` → `AppTopPage` | protected group layout |
| `/app/:organizationId/tickets` | `AppShell` → `OrganizationTicketsPage` | typed `organizationId: string` |
| any other | `NotFoundPage` | catch-all / not-found |

## Typing

- `$organizationId` file naming convention makes TanStack Router generate `organizationId` as `string` in route params.
- No manual zod/valibot validation for params in this issue; that can be added when organization ID format is finalized.

## Testing

Use `@tanstack/react-router` `createMemoryHistory` and `RouterProvider` in tests.

- `/login` and `/signup` render without `AppShell`.
- `/app` renders inside `AppShell`.
- `/app/org-123/tickets` renders inside `AppShell` and displays the `organizationId`.
- `/nonexistent` renders `NotFoundPage`.

## Dependencies

Add to `apps/web`:

- `@tanstack/react-router`
- `@tanstack/router-plugin` (dev dependency, Vite plugin)

`@tanstack/router-devtools` is intentionally omitted to keep the setup minimal; it can be added later if needed.

## Build / Typecheck Changes

- `vite.config.ts`: add `@tanstack/router-plugin/vite` before `@vitejs/plugin-react`.
- `tsconfig.json`: include `src/routeTree.gen.ts` if needed; `allowImportingTsExtensions` already enabled.
- `main.tsx`: replace `<App />` with `<RouterProvider router={router} />`.
- `App.tsx`: remove or repurpose as router entry; existing `tests/unit/App.test.tsx` will be updated to test routing instead.

## Out of Scope

- Authentication guards / `AuthProvider`.
- Real Login, Signup, Ticket UI.
- Data fetching.
